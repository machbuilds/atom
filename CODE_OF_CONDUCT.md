# Code of Conduct

## Our standards

atom is a tool for builders. Communication here is direct, technical, and respectful. We welcome contributors of every background, experience level, identity, and stack.

Examples of behaviour that contribute to a healthy community:

- Specific feedback on code, docs, or design decisions, with reasoning.
- Disagreeing with an idea, not the person who proposed it.
- Asking clarifying questions before assuming intent.
- Crediting prior work when extending it.
- Owning mistakes and fixing them.

Examples of unacceptable behaviour:

- Personal attacks, insults, or derogatory comments.
- Harassment of any kind, public or private.
- Publishing others' personal information without their explicit permission.
- Conduct that a reasonable person would consider inappropriate in a professional setting.

## Enforcement responsibilities

Project maintainers are responsible for clarifying these standards and are expected to take appropriate, fair action in response to any behaviour they deem inappropriate, threatening, offensive, or harmful.

Maintainers have the right to remove, edit, or reject comments, commits, code, issues, and other contributions that don't align with this Code of Conduct.

## Scope

This Code applies in all project spaces — issues, pull requests, discussions, code review comments, and any official atom communication channel — and when an individual is representing the project in public.

## Reporting

To report unacceptable behaviour, open a private security advisory on the [atom repo](https://github.com/mach273/atom/security/advisories/new) or email the maintainer (see `SECURITY.md` for current contact). All complaints will be reviewed promptly and confidentially.
